## Why

管理端已有主题色、圆角、内容布局和菜单风格等外观偏好入口，但这些能力分散在弹窗、设置页和多个本地 store 中，配置项覆盖不完整，语义上也容易和“系统级配置”混淆。

本变更将管理端外观配置明确为“当前用户在当前浏览器中的本地偏好”，提供更完整的可配置项，并保证刷新页面后恢复偏好，不影响用户端。

## What Changes

- 新增管理端外观偏好能力，统一管理主题色、明暗模式、圆角、内容宽度、菜单风格等配置。
- 扩展可配置项，增加字体、界面密度、侧边栏默认状态、动效偏好、表格斑马纹、面包屑显示、页面标题显示等管理端常用外观偏好。
- 将外观偏好保存到浏览器本地，并在管理端应用启动、刷新或重新进入时恢复。
- 梳理现有设置页和 `custom-theme` 弹窗的关系，避免同类外观配置在多个入口中语义冲突。
- 不新增后端系统配置表，不新增系统配置接口，不做跨设备同步。
- 用户端暂不接入该配置，管理端偏好不得影响用户端页面。

## Capabilities

### New Capabilities

- `admin-appearance-preferences`: 管理端用户可配置并本地持久化自己的外观、主题和界面偏好。

### Modified Capabilities

- 无。

## Impact

- 前端 store：整合或扩展 `src/stores/theme.ts`、`src/stores/sidebar-config.ts` 中的管理端本地偏好。
- 前端组件：调整 `src/components/custom-theme/*` 和 `src/pages/settings/components/appearance-form.vue` 的职责边界。
- 前端页面：新增或改造管理端外观偏好配置页，并接入系统管理导航。
- 前端样式：根据偏好更新 `html` class、CSS 变量、布局 class 和局部 UI 显示行为。
- 测试：补充 store 持久化、偏好应用逻辑、配置表单校验和关键页面渲染测试。
- 后端：无接口和数据库变更。
