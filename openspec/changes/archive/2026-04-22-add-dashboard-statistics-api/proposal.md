## Why

当前 Dashboard 仅做静态展示，无法反映后台真实运行与业务状态，容易让管理端首页变成“装饰页”而不是决策入口。现在需要将首页指标接入后端聚合统计接口，形成可复用、可校验、可扩展的数据契约。

## What Changes

- 新增管理端 Dashboard 统计接口，聚合用户总数、今日新增用户、通知数量、操作日志数量、登录趋势与最近操作日志。
- 前端 Dashboard 改为调用后端接口渲染数据，并处理加载、空数据与接口失败状态。
- 后端统计模型应明确时间口径、权限边界与返回字段，避免前端硬编码静态数字。
- 最近操作日志和操作日志数量优先复用审计日志能力，通知数量优先复用系统通知能力。
- 不引入面向普通用户的公开统计接口；该能力仅面向已授权管理员。

## Capabilities

### New Capabilities
- `admin-dashboard-statistics`: 管理端首页统计聚合能力，覆盖概览指标、登录趋势、最近操作日志与前端 Dashboard 数据消费契约。

### Modified Capabilities
- `observability-ops-panel`: 明确 Dashboard 首页业务统计与运维监控面板的边界，运维面板不承担用户/通知/文件/审计日志的首页聚合职责。

## Impact

- 后端：新增或扩展管理端 Dashboard Controller、Service、DTO；读取用户、通知、登录记录、审计日志等数据源。
- 前端：Dashboard 页面、接口请求封装、类型定义、图表/列表数据绑定与异常状态。
- API：新增管理员权限保护的 Dashboard 聚合接口，响应结构需稳定支持卡片、趋势图和日志列表。
- 数据与安全：涉及用户、通知、登录、审计日志统计；需按管理员权限访问，并对日志中的敏感信息做展示级过滤。
