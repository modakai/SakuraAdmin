## Why

目前缺少面向用户端（非管理端）的注册入口与约束，导致注册流程与“只能注册普通用户（user 身份）”这一业务规则难以被统一实现与验收。补齐用户端注册页面与后端强约束，有助于降低误注册、越权注册的风险，并为后续登录/个人中心等用户体系能力打基础。

## What Changes

- 新增用户端注册页面与交互：输入账号信息、校验、提交、成功/失败反馈。
- 新增/调整用户端注册 API：仅允许创建 `user` 身份用户，忽略或拒绝任何客户端传入的角色字段。
- 增加注册相关校验规则与错误码/错误提示映射（前端展示友好文案，后端提供稳定错误标识）。
- 增加最小化的安全与滥用防护预留（如：基础频率限制/验证码能力的接口预留或开关位，具体以现有项目能力为准）。

## Capabilities

### New Capabilities

- `user-registration`: 用户端注册能力（页面 + API + 仅允许注册 user 身份 + 基础校验与错误处理）。

## Impact

- 前端：新增路由与页面（用户端），新增 API 调用与表单校验逻辑，可能新增状态管理/请求封装的调用。
- 后端：新增或调整注册接口与用户创建逻辑，强制角色为 `user`；可能涉及数据库字段默认值/约束、密码加密策略、唯一性校验等。
- 测试：新增前端基础交互测试/后端接口测试（至少覆盖“角色字段被忽略/拒绝”与常见校验失败场景）。
